The entities of your game

Game Objects are the entities of the game, they can be created within a scene and can interact with one another. Game Objects derive from a number of different classes that give them the ability to be updatable, drawable inputable, alarmable, and collidable.

Of these classes, the only two that are required are Updatable and Drawable. Below we have a basic header setup for a game object

// Forward declarations
class GraphicsObject_TextureFlat;
class Matrix;
class Vect;
class Frigate : public GameObject {
public:
Frigate();
~Frigate();
virtual void Update() override;
virtual void Draw() override;
private:
Vect shipPosition;
Matrix shipRotation;
Matrix shipScale;
GraphicsObject_TextureFlat* FrigateGraphicObject;
}

From here, setting up the game object is straightforward:

... // assuming the correct includes
Frigate::Frigate() {
FrigateGraphicObject = new GraphicsObject_TextureFlat(ModelManager::Get("FrigateModel"), ShaderManager::Get("TextureFlatRender"), TextureManager::Get("FrigateTexture");
shipPosition = Matrix(TRANS, 0, 10, 0); // Set the position slightly off the ground
shipRotation = Matrix(IDENTITY);
shipScale = Matrix(IDENTITY);
// Register to allow for updating and drawing of the frigate
Updatable::SubmitUpdateRegistration();
Drawable::SubmitDrawRegistration();
}
Frigate::Update() {
shipPosition += Vect(0, 0, 1) * shipRotation;
Matrix newWorld = shipScale * shipRotation * Matrix(TRANS, shipPosition);
FrigateGraphicObject->SetWorld(newWorld);
}
Frigate::Draw() {
FrigateGraphicObject->Render(SceneManager::GetCurrentScene()->GetCamera());
}
Frigate::~Frigate {
delete FrigateGraphiObject;
}
