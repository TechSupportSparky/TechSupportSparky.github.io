The Terrain Object is a terrain that takes a heightmap and texture as input to generate an engine maintained terrain. An example of loading a terrain object is as follows:

#include "TerrainObjectManager.h"
Sly::LoadAllResources() {
TextureManager::Load("GrassTexture", "grass.tga");
TerrainObjectManager::Load("TerrainName", "HeightMapFile.tga", "GrassTexture", 1000, 80, 1, 1);
}

The above line of code will generate a terrain of size 1000 and height of 80 (determined by the heightmap data gathered from HeightMapFile.tga). The TerrainName may be anything that the coder would like to use. The TextureNameKey is a string that matches an existing texture key within the TextureManager. Note, this does mean that the TerrainObjectManager Load command must come after a texture load. If you attempt to load it before the texture, you will get an error stating the texture does not exist.

The last parameter refers to the UV coordinates. If you are unfamiliar with UV coordinates, think about them as "amount of repeats" in the x and z directions.
