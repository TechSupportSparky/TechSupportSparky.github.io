\doxysubsection{Detailed Description}
How to setup and initialize the developer console. 

The \mbox{\hyperlink{class_sly}{Sly}} \mbox{\hyperlink{class_console}{Console}} allows developers to execute runtime commands in real time. This ability allows for rapid prototyping of new features, heightened debugging potential, as well as user-\/level control over scene features.

The \mbox{\hyperlink{class_sly}{Sly}} \mbox{\hyperlink{class_console}{Console}} functions as an interface for a user-\/level \mbox{\hyperlink{class_game_object}{Game\+Object}} to interface with the \mbox{\hyperlink{class_sly}{Sly}} Engine and its game loop. With this user-\/level \mbox{\hyperlink{class_game_object}{Game\+Object}}, developers may choose to have as much or as little control over the game as they choose to provide to the player with strict definitions of commands.

By default, the \mbox{\hyperlink{class_sly}{Sly}} \mbox{\hyperlink{class_console}{Console}} comes with a handful of built-\/in commands that may be implemented into the developers game object. These built-\/in commands are as follows\+:
\begin{DoxyItemize}
\item \mbox{\hyperlink{class_god_cam_command}{God Cam}}
\item \mbox{\hyperlink{class_kill_command}{Kill}}
\item \mbox{\hyperlink{class_ping_command}{Ping}}
\item \mbox{\hyperlink{class_reload_command}{Reload}}
\item \mbox{\hyperlink{class_time_request_command}{Time}}
\end{DoxyItemize}

Since the \mbox{\hyperlink{class_sly}{Sly}} \mbox{\hyperlink{class_console}{Console}} is expandable, there are unlimited possibilities for further commands that the developer can implement to improve their development process. ~\newline
